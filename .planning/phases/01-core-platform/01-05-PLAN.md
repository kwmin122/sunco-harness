  <behavior>
    **defineSkill:**
    - defineSkill({id: 'harness.init', command: 'init', kind: 'deterministic', stage: 'stable', category: 'harness', routing: 'directExec', description: 'test', execute: async () => ({success: true})}) => returns validated SkillDefinition
    - defineSkill({}) => throws with Zod error listing all missing required fields
    - defineSkill({id: '', ...rest}) => throws because id is empty string
    - defineSkill with valid optional fields (options array) => includes them in result

    **scanner:**
    - scanSkillFiles(['packages/skills-harness/src']) finds all *.skill.ts files
    - scanSkillFiles with empty directory => returns empty array
    - scanSkillFiles with non-existent path => returns empty array (no throw)

    **resolver (D-13 pipeline):**
    - resolveActiveSkills with preset='none', add=[], remove=[] => empty Set
    - resolveActiveSkills with preset='harness', skills discovered => expands preset, filters by discovered
    - resolveActiveSkills with add=['workflow.status'] => includes that skill
    - resolveActiveSkills with remove=['harness.agents'] => excludes that skill
    - resolveActiveSkills with duplicate id in discovered => throws DuplicateSkillError
    - resolveActiveSkills with duplicate command in discovered => throws DuplicateSkillError

    **context:**
    - createSkillContext for deterministic skill => ctx.agent.run() throws "Agent access blocked for deterministic skill"
    - createSkillContext for prompt skill => ctx.agent.run is callable (passes through to real router)
    - ctx.run('other-skill') executes other skill through registry
    - ctx.run('self-id') when self-id is on call stack => throws CircularSkillInvocationError
    - ctx.run('A') where A calls ctx.run('B') where B calls ctx.run('A') => throws CircularSkillInvocationError

    **preset:**
    - expandPreset('none') => empty array
    - expandPreset('harness') => returns harness skill IDs
    - expandPreset('unknown') => empty array (not found, no throw)
  </behavior>
  <implementation>
    **define.ts:**
    - Create Zod schema for SkillDefinition metadata: SkillDefinitionSchema
    - defineSkill<T>(input): SkillDefinition<T> -- validate with Zod, return frozen object
    - Required fields: id (non-empty string), command (non-empty string), kind, stage, category, routing, description, execute (function)

    **scanner.ts:**
    - Import glob from 'glob'
    - scanSkillFiles(basePaths: string[]): scan each basePath for `*.skill.ts` pattern
    - For each file found, dynamically import it and extract the default export
    - Return array of SkillDefinition objects (filter out invalid ones, log warnings)
    - NOTE: In bundled production, scanner reads from a pre-built manifest. For Phase 1, dynamic import is fine.

    **registry.ts:**
    - SkillRegistry class with Map<string, SkillDefinition>
    - register(skill): add to map, check for duplicate id/command (D-14)
    - get(id): return skill or undefined
    - getByCommand(command): find by command field
    - getAll(): return all skills
    - execute(id, context, options): look up skill, call skill.execute(context, options)

    **resolver.ts:**
    - resolveActiveSkills(discovered: SkillDefinition[], policy: SkillPolicyConfig): Set<string>
    - Step 1: Validate all discovered skill metadata
    - Step 2: expandPreset(policy.preset) to get base skill IDs
    - Step 3: Union with policy.add
    - Step 4: Subtract policy.remove
    - Step 5: Filter to only skills present in discovered set
    - Step 6: Apply stage filter (internal skills excluded unless explicitly added)
    - Step 7: Check for duplicate IDs and commands (D-14 fail-fast)
    - Return Set<string> of active skill IDs

    **context.ts:**
    - createSkillContext(params: { skillId, config, stateEngine, agentRouter, uiAdapter, recommender, registry, cwd, callStack? }): SkillContext
    - For deterministic skills (check registry kind): agent = createBlockedAgentProxy()
    - For prompt/hybrid skills: agent = agentRouter
    - run: (targetId, opts) => check circular (callStack includes targetId?), then registry.execute(targetId, createSkillContext({...params, skillId: targetId, callStack: [...callStack, skillId]}), opts)
    - createBlockedAgentProxy(): returns Proxy that throws on any property access with "Agent access blocked for deterministic skill 'X'"

    **preset.ts:**
    - PRESET_REGISTRY: Record<string, string[]> with predefined presets
    - 'none': []
    - 'harness': ['harness.init', 'harness.lint', 'harness.health', 'harness.agents', 'harness.guard']
    - 'workflow': ['workflow.new', 'workflow.scan', 'workflow.discuss', 'workflow.plan', 'workflow.execute', 'workflow.review', 'workflow.verify', 'workflow.ship']
    - 'full': [...harness, ...workflow, 'core.settings', 'core.status', 'core.next', 'core.progress']
    - expandPreset(name): return PRESET_REGISTRY[name] ?? []

    **index.ts:** Re-export all public APIs
  </implementation>
