    **ESLint Runner** (packages/skills-harness/src/lint/runner.ts):
    Per research Pattern 2 and pitfall #2 (flat config only):

    - Export `async function runLint(opts: { files: string[]; boundariesConfig: BoundariesConfig; fix?: boolean; cwd?: string }): Promise<LintResult>`
    - Implementation:
      1. Import `ESLint` from 'eslint' and `boundaries` from 'eslint-plugin-boundaries'.
      2. Import `parser` from 'typescript-eslint' for TypeScript file parsing.
      3. Create ESLint instance:
         ```typescript
         const eslint = new ESLint({
           overrideConfigFile: true,  // bypass filesystem config search
           overrideConfig: [
             {
               files: ['**/*.ts', '**/*.tsx', '**/*.js', '**/*.jsx'],
               languageOptions: {
                 parser: tseslintParser,
                 parserOptions: { ecmaVersion: 'latest', sourceType: 'module' },
               },
               plugins: { boundaries: boundariesPlugin },
               settings: {
                 'boundaries/elements': opts.boundariesConfig.elements,
               },
               rules: {
                 'boundaries/dependencies': [2, {
                   default: 'disallow',
                   rules: opts.boundariesConfig.dependencyRules,
                 }],
               },
             },
           ],
           fix: opts.fix ?? false,
           cwd: opts.cwd,
         });
         ```
      4. Run `const results = await eslint.lintFiles(opts.files)`.
      5. If `fix`, call `await ESLint.outputFixes(results)`.
      6. Transform ESLint results to SunLintViolation[] -- iterate `results[].messages[]`, map each to SunLintViolation per lint/types.ts.
      7. Return LintResult with violations, filesLinted, errorCount, warningCount, fixApplied.

    - Handle errors: wrap in try/catch. If ESLint throws (e.g., no matching files), return LintResult with 0 files and a warning.
    - Per pitfall #3: element patterns must be relative to cwd. The runner receives patterns already constructed by config-generator from init detection.
    - Per anti-pattern: do NOT spawn eslint CLI. Use programmatic API only.
