Document the SUNCO configuration schema (stored in .planning/config.json):

```json
{
  "mode": "interactive",           // yolo | interactive
  "granularity": "standard",       // coarse | standard | fine
  "profile": "balanced",           // quality | balanced | budget | inherit
  "workflow": {
    "research": true,
    "plan_check": true,
    "verifier": true,
    "auto_advance": false,
    "discuss_mode": "discuss",     // discuss | assumptions
    "skip_discuss": false,
    "lint_gate": true,             // SUNCO-only: mandatory lint after execute
    "blast_radius": true,          // SUNCO-only: graph check before execute
    "health_gate": false           // SUNCO-only: health score minimum
  },
  "git": {
    "branching_strategy": "none",  // none | phase | milestone
    "phase_branch_template": "sunco/phase-{phase}-{slug}",
    "milestone_branch_template": "sunco/{milestone}-{slug}"
  },
  "parallelization": { "enabled": true },
  "planning": { "commit_docs": true },
  "hooks": { "context_warnings": true },
  "agent_skills": {}               // per-agent skill injection
}
```

SUNCO-specific settings that GSD doesn't have:
- workflow.lint_gate: mandatory lint after every execute
- workflow.blast_radius: dependency graph check before execution
- workflow.health_gate: minimum health score to proceed
  </action>

    - Default values match GSD behavior where applicable
