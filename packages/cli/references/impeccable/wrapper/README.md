## Why wrapper-not-patch (spec R5)

Impeccable is Apache-2.0. SUNCO could fork + patch. That path was rejected at spec time:

1. **Upstream sync is merge-heavy.** Every upstream release means resolving conflicts on SUNCO-local patches.
2. **License compliance is murkier.** Patching creates a "modified copy" — derivative attribution gets complicated.
3. **Dirty-source risk.** Future readers may miss that vendored source has been modified.

Wrapper-not-patch solves all three: upstream stays pristine, SUNCO adaptations are clearly separate, re-vendoring is `rm -rf + clone + copy + diff`.

## Contract — Context injection

**Who calls**: `/sunco:ui-phase --surface web` (Phase 40/M2.3) loads frontend design context before spawning Impeccable skills.

**Surface**:

```javascript
import { loadDesignContext } from '<path>/references/impeccable/wrapper/context-injector.mjs';
const ctx = loadDesignContext(projectRoot);
// ctx === null         → no DESIGN-CONTEXT.md found
// ctx.source           → '.planning/domains/frontend/DESIGN-CONTEXT.md'
// ctx.version          → '1.0-skeleton' (Phase 38) | '1.0' (Phase 39+)
// ctx.populated_in     → 'Phase 39/M2.2' (skeleton flag)
// ctx.raw_markdown     → full file content
// ctx.sections         → null (Phase 38) | { audience, useCases, brand } (Phase 39+)
```

**Phase 38 behavior**: skeleton — returns raw markdown + sentinel metadata.
**Phase 39/M2.2**: populates `sections` by parsing DESIGN-CONTEXT.md structure generated by `/sunco:discuss --domain frontend`.

**Smoke test**: `node context-injector.mjs --test` builds a temp fixture, exercises the loader in both present-and-absent cases, and asserts the contract shape.

## Fallback policy (detector-unavailable) — G8

**Contract** (documented here, executable in Phase 41/M2.4):

If the vendored detector cannot run (Node unavailable, dependency load failure, runtime error, browser-only environment), `/sunco:ui-review` MUST:

1. Throw or surface `DetectorUnavailableError` (exported from `detector-adapter.mjs`) with a `cause` describing the failure.
2. Log a warning to stderr: `⚠ Impeccable detector unavailable — <reason>. Continuing with LLM-only critique.`
3. Write a partial `IMPECCABLE-AUDIT.md` containing:
   - The warning banner
   - LLM-only critique section (generated by the calling workflow)
   - An explicit "detector skipped" marker so downstream agents know findings are incomplete
4. Exit cleanly — do **not** hard-fail the whole workflow.

**No silent failure.** The fallback path is visible in output and in the audit artifact.

**Executable fallback** lands in Phase 41/M2.4 (per Gate 2 G8 scope trim). Phase 38 provides only the `DetectorUnavailableError` export + this documentation contract.

## Pristine invariant (G4)

The `references/impeccable/source/` and `references/impeccable/src/` directories are **byte-identical** to upstream@`00d485659af82982aef0328d0419c49a2716d123`.

**Verification command** (run after re-cloning upstream at the same pinned SHA):

```bash
cd ~/SUN/sunco-harness
diff -r tmp/impeccable-upstream/source/skills packages/cli/references/impeccable/source/skills
diff -q tmp/impeccable-upstream/src/detect-antipatterns.mjs packages/cli/references/impeccable/src/detect-antipatterns.mjs
diff -q tmp/impeccable-upstream/LICENSE packages/cli/references/impeccable/LICENSE
diff -q tmp/impeccable-upstream/NOTICE.md packages/cli/references/impeccable/NOTICE.md
# All should return empty. Any output = pristine violation.
```

**Install-pipeline protection**: `packages/cli/bin/install.cjs` uses `copyDirRecursive` (not `copyDirWithReplacement`) for `references/impeccable/source/` and `references/impeccable/src/`. This prevents non-Claude runtime installs (codex/cursor/etc.) from rewriting upstream string literals — specifically `.claude/skills` inside `source/skills/impeccable/scripts/cleanup-deprecated.mjs`, and `.impeccable.md` references in upstream SKILL.md files. Automated enforcement lives in `packages/cli/bin/smoke-test.cjs` Section 13.
