### Task 1: Artifact Summarizer
**read_first**: packages/skills-workflow/src/shared/phase-reader.ts
**files**: packages/skills-workflow/src/shared/artifact-summarizer.ts (NEW)

Create deterministic artifact summarizer (zero LLM cost):

```typescript
export interface ArtifactSummary {
  phaseNumber: number
  filename: string
  summary: string        // 3-line max
  keyDecisions: string[] // extracted D-NNN entries
  status: string         // from frontmatter or heading
  originalTokens: number // estimated via chars/4
  summaryTokens: number  // estimated via chars/4
  reductionPercent: number
}

export function summarizeArtifact(content: string, phaseNumber: number, filename: string): ArtifactSummary
```

Summarization algorithm (deterministic):
1. Extract first heading (# line) as title
2. Extract status/goal from first paragraph (truncate at 200 chars)
3. Extract D-NNN decision entries via regex
4. Count original vs summary tokens (chars/4 estimate)
5. Calculate reduction percentage

**acceptance_criteria**:
- GREPPABLE: `summarizeArtifact` exported from artifact-summarizer.ts
- Unit test: CONTEXT.md input → 3-line summary + decisions extracted
- Unit test: reduction percentage >= 70% for typical 2KB artifact

### Task 2: Selective Phase Reader Wrapper
**read_first**: packages/skills-workflow/src/shared/phase-reader.ts, packages/skills-workflow/src/shared/context-zones.ts
**files**: packages/skills-workflow/src/shared/phase-reader.ts

Add new function (don't modify existing readPhaseArtifact):

```typescript
export async function readPhaseArtifactSmart(
  cwd: string,
  phaseNumber: number,
  filename: string,
  opts: {
    currentPhase: number,
    contextZone: ContextZone,
  }
): Promise<{ content: string | null, mode: 'full' | 'summary', summary?: ArtifactSummary }>
```

Loading rules:
- Current phase or Green zone: full load (return original content)
- Completed phase + Yellow zone: full load (still space)
- Completed phase + Orange zone: summary only
- Completed phase + Red zone: skip (return null)
- Always full load for current phase regardless of zone

**acceptance_criteria**:
- GREPPABLE: `readPhaseArtifactSmart` exported from phase-reader.ts
- Unit test: completed phase + orange zone → summary mode
- Unit test: current phase + red zone → full mode (always full for current)
