# Roadmap

Generated by `/sunco:new`. Phases are ordered and independently deliverable — each phase produces a working artifact, not just intermediate code. Every phase has a single clear owner, a completion date estimate, and an explicit dependency chain so agents never execute work out of order.

---

## How to Read This Roadmap

**Phase status lifecycle:**
- `not started` → phase is planned but work has not begun
- `in progress` → active execution underway (plans written, code being written)
- `executing` → agent is actively running tasks inside the phase
- `verifying` → all tasks complete, verification pipeline running
- `complete` → exit criteria met, verified, and committed
- `blocked` → a blocker must be resolved before execution can resume
- `deferred` → intentionally postponed to a later milestone

**Complexity ratings:**
- `simple` — straightforward implementation, 1–2 files changed, no cross-cutting concerns, < 4 hours
- `moderate` — multiple files or subsystems involved, some design judgment required, 4–16 hours
- `complex` — architectural impact, cross-cutting concerns, performance sensitivity, or significant unknowns, 16+ hours

**Dependency notation:**
- `depends on: none` — first in chain, can start immediately
- `depends on: Phase N` — requires Phase N to be `complete` before starting
- `depends on: Phase N (partial)` — can start after Phase N milestone X is reached, before Phase N is fully complete
- `depends on: Phase N, Phase M` — requires both phases complete (parallel convergence point)

## Versioning Convention

This project uses semantic versioning: `MAJOR.MINOR.PATCH`

- `MAJOR` — milestone boundary (breaking change or fundamental architecture shift)
- `MINOR` — phase boundary (new capability delivered)
- `PATCH` — hotfix or correctional work within a phase
