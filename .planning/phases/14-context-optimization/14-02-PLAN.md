Create `code-graph.ts` with a CodeGraph class. NO external dependencies (no Tree-sitter, no NetworkX). Pure regex + Map-based graph.

```typescript
export interface GraphNode {
  file: string;         // relative path
  exports: string[];    // exported symbols
  imports: Array<{ from: string; symbols: string[] }>;
}

export interface GraphEdge {
  from: string;  // importer file
  to: string;    // imported file (resolved)
}

export interface BlastRadiusResult {
  directDeps: string[];     // files that directly import the changed file
  transitiveDeps: string[]; // files affected transitively (2+ hops)
  totalAffected: number;
}

export class CodeGraph {
  private nodes: Map<string, GraphNode> = new Map();
  private edges: GraphEdge[] = [];
  // Reverse dependency map: file -> files that import it
  private reverseDeps: Map<string, Set<string>> = new Map();

  // Build graph from file list
  async build(files: string[], basePath: string): Promise<void>

  // Parse imports from a single file (regex-based, supports TS/JS/Python/Go)
  parseImports(content: string, filePath: string): Array<{ from: string; symbols: string[] }>

  // Resolve import path to actual file
  resolveImport(importPath: string, fromFile: string, basePath: string): string | null

  // Get blast radius for changed files
  blastRadius(changedFiles: string[], maxDepth?: number): BlastRadiusResult

  // Serialize/deserialize for persistence
  toJSON(): { nodes: GraphNode[]; edges: GraphEdge[] }
  static fromJSON(data: { nodes: GraphNode[]; edges: GraphEdge[] }): CodeGraph

  // Stats
  get nodeCount(): number
  get edgeCount(): number
}
```

Import parsing regexes (cover 80% of cases):
- TypeScript/JavaScript: `import { ... } from '...'`, `import ... from '...'`, `import '...'`, `require('...')`
- Python: `from ... import ...`, `import ...`
- Go: `import "..."`, `import ( "..." )`

Blast radius algorithm:
1. Start with changedFiles set
2. BFS through reverseDeps
3. Track depth (direct = depth 1, transitive = depth 2+)
4. Cap at maxDepth (default 5) to prevent explosion
5. Return deduplicated sorted lists

File scanning:
- Walk directory recursively (node:fs/promises readdir + stat)
- Filter by extension: .ts, .tsx, .js, .jsx, .py, .go
- Ignore: node_modules, dist, .git, .sun, __pycache__
- Cap at 5000 files (large repos)

Tests (10+):
- parseImports: TS named import, default import, require, Python from/import, Go import
- resolveImport: relative path (./ ../), package name (returns null)
- blastRadius: direct dep, transitive 2-hop, cycle handling, empty graph
- toJSON/fromJSON round-trip
- build from real file content
