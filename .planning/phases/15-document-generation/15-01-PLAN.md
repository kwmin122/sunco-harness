Create HwpxWriter class that generates valid HWPX files.

HWPX structure (minimal viable):
```
document.hwpx (ZIP file containing):
├── META-INF/
│   └── manifest.xml    (file manifest)
├── Contents/
│   ├── content.hpf     (content manifest)
│   ├── header.xml      (document header — page settings)
│   └── section0.xml    (body content — paragraphs, tables)
└── mimetype            (application/hwp+zip)
```

Use `node:zlib` (gzip) — actually HWPX is a ZIP file. Use the built-in approach:
- Create individual XML files as strings
- Use a minimal ZIP implementation (store method, no compression needed for small docs)
- OR use `child_process` to call system `zip` command
- BEST: write a tiny ZIP creator (~80 lines) that creates uncompressed ZIP files
  - ZIP local file header + central directory + end of central directory
  - All stored (compression method 0) — simplest, works for text XML

HwpxWriter API:
```typescript
class HwpxWriter {
  private sections: string[] = [];

  addParagraph(text: string, style?: 'title' | 'heading1' | 'heading2' | 'body'): void
  addTable(headers: string[], rows: string[][]): void
  addPageBreak(): void

  // Generate the .hwpx ZIP buffer
  async generate(): Promise<Buffer>

  // Write to file
  async writeTo(filePath: string): Promise<void>
}
```

section0.xml format (simplified OWPML):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<hs:sec xmlns:hs="http://www.hancom.co.kr/hwpml/2011/paragraph">
  <hs:p>
    <hs:run>
      <hs:t>paragraph text here</hs:t>
    </hs:run>
  </hs:p>
</hs:sec>
```

Tests (8+):
- addParagraph generates valid XML
- addTable generates table XML
- generate returns Buffer (check ZIP magic bytes PK\x03\x04)
- writeTo creates file on disk
- Empty document generates valid structure
- Title style vs body style
- Multiple sections
- Round-trip: write then check file exists and has content

Flow:
1. Read project context (PROJECT.md, ROADMAP.md, package.json, README if exists)
2. Build prompt with buildDocPrompt (type, context, format)
3. Dispatch agent to generate content
4. If --hwpx: parse agent output → HwpxWriter → .hwpx file
5. If --md: write agent output as .md file
6. If --template: read template, fill {{placeholders}} with agent-generated content

buildDocPrompt should instruct agent to output structured content with section markers that can be parsed into HwpxWriter calls.

Template system:
- Read .sun/templates/{name}.md
- Extract {{placeholder}} patterns
- Agent generates values for each placeholder
- String replacement → output file
