       Export `buildResearchPrompt(topic: string, idea: string, context: Record<string, string>): string`

       The function builds a prompt for a research agent. Topics are things like:
       - 'tech-stack': "Research the best technology stack for this project"
       - 'competitors': "Research existing solutions and competitors"
       - 'architecture': "Research architecture patterns suitable for this project"
       - 'challenges': "Research common challenges and pitfalls for this type of project"
       - 'ecosystem': "Research the ecosystem, libraries, and tools available"

       Prompt structure:
       ```
       You are a technical researcher. Your task is to research ONE specific topic for a new project.

       ## Project Idea
       {idea}

       ## Context (from user answers)
       {formatted context key-value pairs}

       ## Research Topic: {topic}

       {topic-specific instructions}

       ## Output Format
       Produce a concise research report (500-1000 words) with:
       - Key findings (bullet points)
       - Recommendations with rationale
       - Risks or concerns
       - Sources or references where applicable

       Be specific and actionable. Avoid generic advice.
       ```

       The topic-specific instructions vary:
       - tech-stack: "Identify languages, frameworks, databases, and infrastructure. Consider the project's scale, target platform, and team constraints from context."
       - competitors: "Find existing products, open-source projects, and approaches in this space. Note what they do well and what gaps exist."
       - architecture: "Propose architecture patterns (monolith, microservices, serverless, etc.) with trade-offs for this specific project."
       - challenges: "List technical and non-technical challenges. For each, suggest mitigation strategies."
       - ecosystem: "Map the relevant ecosystem: libraries, tools, hosting, CI/CD, monitoring. Recommend specific packages with version numbers."

    2. Execute function flow (per D-01):

       **Step 1 - Get idea (D-01 step 1):**
       ```typescript
       const positional = (ctx.args._ as string[] | undefined) ?? [];
       let idea = positional.join(' ').trim();
       if (!idea) {
         const response = await ctx.ui.askText({
           message: 'Describe your project idea:',
           placeholder: 'A CLI tool that...',
         });
         idea = response.text;
       }
       if (!idea) {
         await ctx.ui.result({ success: false, title: 'New Project', summary: 'No idea provided' });
         return { success: false, summary: 'No idea provided' };
       }
       ```

       **Step 1.5 - Check provider availability (D-06):**
       Check ctx.agent.listProviders(). If empty, show error result per research Pattern 4.

       **Step 2 - Ask 5-8 clarifying questions (D-01 step 2, D-04):**
       Define a question tree with conditional branching. Use ctx.ui.ask() for choice questions and ctx.ui.askText() for freeform. Track answers in a Record<string, string>.

       Questions (adaptive per D-04):
       1. "What type of project?" -- choices: CLI tool, Web app, API service, Library, Mobile app, Other
       2. "Target platform?" -- choices derived from Q1 answer (e.g., Web -> Browser/Node/Both)
       3. "Primary language?" -- choices: TypeScript, Python, Rust, Go, Java, Other (use askText for Other)
       4. "Describe your target users:" -- freeform askText
       5. "What's the core problem you're solving?" -- freeform askText
       6. "Scale expectations?" -- choices: Personal/side project, Small team, Startup, Enterprise
       7. (conditional) If Web: "Frontend framework preference?" -- choices
       8. (conditional) If API: "Database preference?" -- choices

       Show progress during questions: ctx.ui.progress({ title: 'Gathering context', total: questionCount })

       **Step 3 - Parallel research dispatch (D-01 step 3, D-02):**
       ```typescript
       const researchProgress = ctx.ui.progress({ title: 'Researching', total: RESEARCH_TOPICS.length });
       const researchResults = await Promise.allSettled(
         RESEARCH_TOPICS.map(topic =>
           ctx.agent.run({
             role: 'research',
             prompt: buildResearchPrompt(topic, idea, answers),
             permissions: {
               role: 'research', readPaths: ['**'], writePaths: [],
               allowTests: false, allowNetwork: false, allowGitWrite: false, allowCommands: [],
             },
             timeout: 120_000,
           })
         )
       );
       ```

       Collect successful results, log warnings for failures. Update progress per completed agent.

       **Step 4 - Synthesis via planning agent (D-01 step 4, D-05):**
       ```typescript
       const successfulResearch = researchResults
         .map((r, i) => r.status === 'fulfilled' && r.value.success
           ? { topic: RESEARCH_TOPICS[i], content: r.value.outputText }
           : null)
         .filter(Boolean);

       const synthesisResult = await ctx.agent.run({
         role: 'planning',
         prompt: buildSynthesisPrompt(idea, answers, successfulResearch),
         permissions: {
           role: 'planning', readPaths: ['**'], writePaths: ['.planning/**'],
           allowTests: false, allowNetwork: false, allowGitWrite: false, allowCommands: [],
         },
         timeout: 180_000,
       });
       ```

       **Step 5 - Write artifacts (D-03):**
       Parse synthesis output by splitting on `---DOCUMENT_SEPARATOR---`.
       Write each document via writePlanningArtifact(ctx.cwd, filename, content).
       If parsing fails (separator not found), write the entire output as PROJECT.md and warn.

       Return success with details listing created files.
