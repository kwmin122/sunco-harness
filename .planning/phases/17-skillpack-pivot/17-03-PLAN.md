**new.md** — Bootstrap project from idea:
  - Questions until idea fully understood
  - Optional parallel research agents (4x)
  - Requirements extraction (v1/v2/out-of-scope)
  - Roadmap creation with phase mapping
  - Creates: PROJECT.md, REQUIREMENTS.md, ROADMAP.md, STATE.md
  - After: /sunco:discuss 1

**discuss.md** — Extract decisions for a phase:
  - Read ROADMAP.md + any existing CONTEXT
  - Identify gray areas by feature type
  - Ask user for preferences
  - Support --batch for grouped questions
  - Support discuss mode vs assumptions mode (reads codebase first)
  - Creates: {N}-CONTEXT.md
  - After: /sunco:plan N

**plan.md** — Create execution plans:
  - Read CONTEXT.md for the phase
  - Research phase implementation (optional, toggle)
  - Create 2-3 atomic task plans with XML structure
  - Plan checker verifies against requirements (loop max 3x)
  - Creates: {N}-RESEARCH.md, {N}-{M}-PLAN.md
  - After: /sunco:execute N

**execute.md** — Execute plans in parallel waves:
  - Group plans by dependency into waves
  - Spawn Agent per plan (fresh context)
  - Atomic commits per task
  - **SUNCO diff**: mandatory lint-gate after each plan (harness.lint)
  - **SUNCO diff**: blast radius check before execution (graph)
  - Creates: {N}-{M}-SUMMARY.md, {N}-VERIFICATION.md
  - After: /sunco:verify N

**verify.md** — 6-layer Swiss cheese verification:
  - Layer 1: Multi-agent cross-review
  - Layer 2: Guardrails (run /sunco:lint + /sunco:health via engine)
  - Layer 3: BDD criteria check against PLAN acceptance criteria
  - Layer 4: Permission audit (file access, network, git)
  - Layer 5: Adversarial test
  - Layer 6: Cross-model verification (Codex) — if codex plugin installed,
    run /codex:adversarial-review automatically. Eliminates same-model bias.
    If not installed, skip gracefully with note.
    Install: /plugin marketplace add openai/codex-plugin-cc && /plugin install codex@openai-codex
  - Creates: {N}-VERIFICATION.md
  - After: /sunco:ship N or fix

**review.md** — Multi-provider cross-review:
  - Creates REVIEWS.md with findings
  - After: address findings or ship

**ship.md** — Create PR with verification pre-check:
  - Run verify first
  - Generate PR body from SUMMARYs
  - Create PR via gh
  - Support --draft

**auto.md** — Full autonomous pipeline:
  - Loop: discuss → plan → execute → verify per phase
  - Crash recovery (AutoLock)
  - Stuck detection
  - Budget ceiling
  - Adaptive replan (re-read ROADMAP after each phase)
  - **SUNCO diff**: mandatory lint-gate in pipeline
