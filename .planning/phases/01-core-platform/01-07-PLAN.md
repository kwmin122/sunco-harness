    **InkUiAdapter.mountPattern implementation:**
    Use Ink's render() API to mount pattern components and wait for completion.

    - kind='entry': render SkillEntry component, resolve after brief display (~500ms or immediate)
    - kind='choice': render InteractiveChoice component, resolve with UiChoiceResult when user selects
    - kind='progress': render SkillProgress component, return handle ID for update/dispose
    - kind='result': render SkillResult component, resolve after display

    **Key implementation detail:**
    Ink's render() returns an instance with unmount(), waitUntilExit(), rerender(). Use this to:
    1. Mount pattern component
    2. Wait for user interaction (choice) or display completion (entry/result)
    3. Unmount and resolve

    For progress: maintain a Map<string, InkInstance> of active progress renders. update() calls rerender() with new props. dispose() calls unmount().

    **Pattern for choice:**
    ```typescript
    async mountChoice(data: AskInput): Promise<UiOutcome> {
      return new Promise((resolve) => {
        const { unmount } = render(
          React.createElement(InteractiveChoice, {
            ...data,
            onSelect: (result: UiChoiceResult) => {
              unmount();
              resolve({ kind: 'choice', result });
            },
          })
        );
      });
    }
    ```

    Ensure InkUiAdapter imports from the patterns directory, NOT from raw Ink components.
