    - When gh CLI is not available (execa throws), returns success=true with manual fallback instructions
    - When gh CLI is available, calls gh pr create with --title and --body
    - When --draft flag is set, passes --draft to gh pr create
    - PR body includes phase name, verification verdict, changelog summary
    - When --phase flag is provided, uses that phase number for context
    - Creates branch with naming pattern ship/phase-{N}-{slug} if not already on a non-main branch

    **ship.skill.ts** (per D-01 through D-04, D-15, D-16):
    - `defineSkill` with id: 'workflow.ship', command: 'ship', kind: 'prompt' (per Pitfall 8 -- needs agent access for verify), stage: 'stable', category: 'workflow', routing: 'directExec'.
    - Options: `{ flags: '-p, --phase <number>', description: 'Phase to ship' }`, `{ flags: '--skip-verify', description: 'Skip verification pre-check' }`, `{ flags: '--draft', description: 'Create draft PR' }`.
    - Execute flow (per D-01):
      1. Import `captureGitState` from `./shared/git-state.js`. Capture current git state. If dirty tree, show warning but proceed.
      2. **Verify pre-check** (per D-02): Unless `--skip-verify` is set, call `const verifyResult = await ctx.run('workflow.verify')`. If `!verifyResult.success`, extract failed layers from `verifyResult.data` (cast as VerifyReport), return `{ success: false, summary: 'Ship blocked: verification failed ({N} layer(s))', data: { verifyResult } }`.
      3. **Branch creation** (per D-01 step 2): Check current branch via gitState.branch. If on 'main' or 'master', create new branch `ship/phase-{phaseNumber}-{timestamp}` using `simpleGit(ctx.cwd).checkoutLocalBranch()`. Otherwise use current branch.
      4. **Push** (per D-01 step 3): Push branch to origin using `simpleGit(ctx.cwd).push('origin', branchName, ['--set-upstream'])`.
      5. **PR creation** (per D-03): Dynamic import execa: `const { execa } = await import('execa')`. Try `await execa('gh', ['auth', 'status'])` to check auth. If fails, return fallback: `{ success: true, summary: 'Branch pushed. Create PR manually.', warnings: ['GitHub CLI (gh) not available or not authenticated.'], data: { manual: true, branch: branchName } }`.
      6. If gh available: Build PR body via `buildShipPrBody()` from `./prompts/ship-pr-body.js`. Build title: `[Phase {N}] {phaseName}`. Call `execa('gh', ['pr', 'create', '--title', title, '--body', body, ...(draft ? ['--draft'] : [])])`. Parse stdout for PR URL.
      7. Return `{ success: true, summary: 'PR created: {url}', data: { prUrl, branch, verifyVerdict } }`.
    - Use `ctx.ui.entry()` at start and `ctx.ui.result()` at end per established pattern.
    - Use `ctx.ui.progress()` for verify step, branch step, push step, PR step.

  <behavior>
    release.test.ts:
    - Default bump type is 'patch' when no --major/--minor/--patch flag
    - --major flag bumps major version
    - --minor flag bumps minor version
    - Reads current version from root package.json
    - Calls updateAllVersions to update all workspace packages
    - Generates CHANGELOG.md from git log since last tag
    - Creates annotated git tag v{newVersion}
    - Blocks when working tree is dirty (unless --force)
    - When --dry-run, reports what would happen without making changes
    - When --skip-publish, skips npm publish step
    - When tag already exists, warns and skips tagging
    - Pushes tag to remote after creation
  </behavior>
  <action>
    **release.skill.ts** (per D-05 through D-07, D-15, D-16):
    - `defineSkill` with id: 'workflow.release', command: 'release', kind: 'deterministic' (per D-07 -- no agent needed), stage: 'stable', category: 'workflow', routing: 'directExec'.
    - Options: `{ flags: '--major', description: 'Bump major version' }`, `{ flags: '--minor', description: 'Bump minor version' }`, `{ flags: '--patch', description: 'Bump patch version (default)' }`, `{ flags: '--dry-run', description: 'Preview without publishing' }`, `{ flags: '--skip-publish', description: 'Skip npm publish step' }`, `{ flags: '--force', description: 'Proceed even with dirty working tree' }`.
    - Execute flow (per D-05):
      1. **Dirty tree check** (per Pitfall 2): Import `captureGitState`. If `!gitState.isClean && !ctx.args.force`, return `{ success: false, summary: 'Working tree is dirty. Commit changes first or use --force.' }`.
      2. **Determine bump type**: Check `ctx.args.major`, `ctx.args.minor`, `ctx.args.patch` flags. Default to 'patch'.
      3. **Read current version**: `readFile(join(ctx.cwd, 'package.json'), 'utf-8')` -> JSON.parse -> extract version.
      4. **Bump version**: Import `bumpVersion` from `./shared/version-bumper.js`. Compute newVersion.
      5. **Dry-run check**: If `ctx.args['dry-run']` (Commander.js camelCase), report all steps that WOULD happen and return early with `{ success: true, summary: 'Dry run: would bump to {newVersion}', data: { dryRun: true, newVersion } }`.
      6. **Update all packages**: Import `updateAllVersions` from `./shared/version-bumper.js`. Call with cwd and newVersion. Capture list of updated files.
      7. **Generate changelog**: Import `parseGitLog`, `generateChangelog`, `prependChangelog` from `./shared/changelog-writer.js`. Get git log since last tag via `simpleGit(ctx.cwd).log({ from: lastTag, to: 'HEAD' })`. If no tags, use full log. Parse entries, generate section, prepend to CHANGELOG.md. Write via `node:fs/promises writeFile`.
      8. **Git commit**: Stage all changed package.json files + CHANGELOG.md. Commit with message `release: v{newVersion}`.
      9. **Create tag** (per Pitfall 7): Check if tag exists via `simpleGit.tags()`. If `v{newVersion}` exists, warn and skip. Otherwise `simpleGit.addAnnotatedTag('v{newVersion}', 'Release v{newVersion}')`.
      10. **npm publish** (per D-05 step 4): Unless `--skip-publish`, dynamic import execa: `const { execa } = await import('execa')`. Call `execa('npm', ['publish', '--access', 'public'], { cwd: ctx.cwd })`. Wrap in try/catch -- failure should warn, not fail the entire release.
      11. **Push tag**: `simpleGit.push('origin', branchName)` + `simpleGit.pushTags('origin')`.
      12. Return `{ success: true, summary: 'Released v{newVersion}', data: { version: newVersion, updatedFiles, tagCreated, published } }`.

    - Test all behavior cases. Verify version bump logic, dirty tree block, dry-run short circuit, tag conflict handling.
