## 1. GSD Workflow Analysis — Research-First Planning

### GSD의 plan-phase 사이클

```
discuss-phase (CONTEXT.md — 사용자 설계 의견)
  → plan-phase 내부:
    → [Step 5] gsd-phase-researcher → RESEARCH.md (도메인/기술 리서치)
    → [Step 5.5] Nyquist Validation Strategy → VALIDATION.md
    → [Step 5.6] UI Design Contract Gate → UI-SPEC.md (프론트엔드만)
    → [Step 8] gsd-planner → PLAN.md files (BDD + read_first + acceptance_criteria)
    → [Step 10] gsd-plan-checker → VERIFICATION PASSED / ISSUES FOUND
    → [Step 12] Revision Loop (max 3 iterations: planner ↔ checker)
    → [Step 13] Requirements Coverage Gate (모든 REQ-ID가 최소 1개 plan에 매핑)
  → execute-phase (wave-based parallel, gsd-executor subagents)
  → verify-work (gsd-verifier — goal-backward 4-level verification)
```

### SUNCO에 도입할 것

현재 SUNCO의 `sunco plan`은 단일 에이전트 디스패치 + plan-checker 검증 루프를 갖고 있다.
GSD 대비 부족한 점:

| GSD Feature | SUNCO 현재 | Gap |
|-------------|-----------|-----|
| Research step integrated in plan | `sunco research`는 별도 스킬 | plan에 research 자동 통합 필요 |
| Nyquist Validation Strategy | 없음 | VALIDATION.md 생성 로직 추가 |
| Requirements Coverage Gate | 없음 | plan 완료 시 REQ-ID 커버리지 체크 |
| Deep Work Rules (read_first, acceptance_criteria, action) | acceptance_criteria만 | read_first + concrete action 강제 |
| Revision Loop (max 3) | 1회 checker만 | 최대 3회 planner ↔ checker 반복 |
| UI Design Contract Gate | 없음 (Extension 영역 4에 계획) | Phase 인식 기반 UI-SPEC 게이트 |

### 구현 전략

`sunco plan --research` 플래그로 통합:
1. research 서브스킬 자동 호출 → RESEARCH.md 생성
2. RESEARCH.md에서 Validation Architecture 추출 → VALIDATION.md
3. planner 디스패치 (research + context + requirements 입력)
4. plan-checker 검증 (max 3 iterations)
5. requirements coverage gate (plan의 requirements 필드 vs ROADMAP 요구사항)

### 핵심 포지셔닝 차별점

```
GSD = Claude Code 위에 올라가는 스킬 팩 (Claude 종속)
SUNCO = 독립 CLI 런타임 (Provider-agnostic) + 하네스 + 한국어 시장 독점
```

GSD가 못하는 것:
1. **린트/가드/헬스**: 결정적 하네스 없음
2. **Provider-agnostic**: Claude-only (OpenAI Codex, Gemini 불가)
3. **한국어 시장**: 영어만 지원
4. **독립 배포**: Claude Code 없이 동작 불가

SUNCO가 GSD보다 나아야 할 것:
1. **plan 품질**: research → validate → plan → checker loop → requirements coverage
2. **한국 시장 지원**: i18n, 카카오 알림, HyperCLOVA X, 한국어 가이드
3. **독립 런타임**: `npx sunco` 단독 실행

## 4. 제안: v1.1 Phase 구성

- `sunco plan --research` 통합 리서치 스텝

- `sunco gha <url>` (GitHub Actions 실패 분석)

### Phase 15: Skill Marketplace
