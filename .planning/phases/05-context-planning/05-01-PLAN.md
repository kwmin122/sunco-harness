    - Include instruction to suggest 2-4 options per gray area, marking one as recommended

    - Export `buildDiscussDeepDivePrompt(area: { id: string; question: string; userAnswer: string }, priorDecisions: Record<string, string>, phaseGoal: string): string`

    - Output format: `DECISION: {text}\nFOLLOW_UP: {question or "none"}\nCONFLICTS: {list or "none"}`

    - These are acceptance criteria invisible to coding agents, used by verification agents only

    Step 1: Determine phase number. Read ctx.args.phase if provided, else read STATE.md via `readFile(join(ctx.cwd, '.planning', 'STATE.md'), 'utf-8')` and parse with `parseStateMd()` to get current phase. Resolve phase dir with `resolvePhaseDir()`.

    Step 2: Read ROADMAP.md from `.planning/ROADMAP.md`, parse with `parseRoadmap()` to extract phase goal, requirements, and success criteria for the target phase. If ROADMAP.md not found or phase not found, return error.

    Step 3: Agent identifies gray areas. Dispatch agent with `role: 'planning'`, read-only permissions (`writePaths: []`), using `buildDiscussAnalyzePrompt(phaseGoal, requirements, codebaseContext)`. Parse output to extract gray areas using `---GRAY_AREA---` separator. If parsing fails, use the whole output as a single text area and present a simple text question instead.

    Step 4: Present gray areas to user. For each parsed gray area, call `ctx.ui.ask({ message: area.question, options: area.options })`. Accumulate answers in a `decisions: Record<string, string>` map.

    Step 5: Deep-dive each selected area. For each area with answer, dispatch agent with `buildDiscussDeepDivePrompt(area, decisions, phaseGoal)`. Parse the response to extract `DECISION:` lines. Accumulate locked decisions.

    Step 6: Generate holdout scenarios. Dispatch agent with `buildDiscussScenarioPrompt(decisions, phaseGoal, requirements)`. Parse output by `---SCENARIO---` separator. Write each scenario to `.sun/scenarios/` via `ctx.fileStore.write('scenarios', \`scenario-{phaseNumber}-{idx}.md\`, content)`. If this step fails, continue with warnings (D-02 is important but should not block CONTEXT.md creation).

    ```
    {relevant code patterns}
    </code_context>

    <specifics>
    {specific implementation ideas}
    </specifics>

    <deferred>
    {deferred items}
    </deferred>
    ```
