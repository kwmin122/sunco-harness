## Tasks

### Task 1: Context Zone Types + Utility
**read_first**: packages/skills-workflow/src/shared/budget-guard.ts, packages/core/src/agent/tracker.ts
**files**: packages/skills-workflow/src/shared/context-zones.ts (NEW)

Create shared context zone utility:
```typescript
export type ContextZone = 'green' | 'yellow' | 'orange' | 'red'
export interface ContextZoneResult {
  zone: ContextZone
  usedPercent: number
  suggestPause: boolean
  suggestCompact: boolean
  message: string | null
}
export function classifyContextZone(usedPercent: number): ContextZoneResult
```

Thresholds:
- Green: 0-50% → no action
- Yellow: 50-70% → informational
- Orange: 70-85% → suggest /sunco:pause, auto-save HANDOFF
- Red: 85%+ → suggest compact, capture state before auto-compact

**acceptance_criteria**:
- GREPPABLE: `classifyContextZone` exported from context-zones.ts
- GREPPABLE: `ContextZone` type exported
- Unit tests: all 4 zones return correct classification

### Task 2: Enhance Context Monitor Hook
**read_first**: packages/cli/hooks/sunco-context-monitor.cjs
**files**: packages/cli/hooks/sunco-context-monitor.cjs

Extend existing 3-tier thresholds to 4-tier zone system:

Current thresholds (95/85/70) → New zones:
- Red (85%+): CRITICAL — suggest compact, auto-save state
- Orange (70-85%): HIGH — suggest /sunco:pause
- Yellow (50-70%): MODERATE — informational warning
- Green (0-50%): no output (silent)

Add to output:
- Zone color indicator in stderr message
- When Orange+: print `💡 Run /sunco:pause to save context and resume later`
- When Red: print `⚠️  Context critical — auto-compact imminent, saving state...`
- Write `.sun/context-zone.json` with `{ zone, usedPercent, timestamp }` for other skills to read

**acceptance_criteria**:
- GREPPABLE: `context-zone.json` written by hook
- Orange zone (70-85%) shows pause suggestion in stderr
- Red zone (85%+) shows compact warning in stderr
