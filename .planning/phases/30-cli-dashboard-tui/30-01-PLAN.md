    Create `packages/skills-workflow/src/dashboard-tui.tsx`. Exports:

    ```ts
    export async function renderDashboardTui(cwd: string): Promise<void>;
    // Top-level entry. Validates TTY, mounts Ink, polls active-work at 1 Hz,
    // resolves the returned promise when the user exits (q / Q / Ctrl+C / Esc).

    export function formatRelativeTime(iso: string, now?: Date): string;
    // Helper for "2m ago" / "40s ago" / "just now". Exported for unit tests.

    // Helper for "0:48" / "4:12". Exported for unit tests.
    ```

    Implementation rules:

    1. **TTY guard** — at the top of `renderDashboardTui`, check `process.stdout.isTTY`. If falsy:
       ```
       process.stderr.write(
         "sunco status --live requires a TTY. Use 'sunco status' for a one-shot snapshot.\n",
       );
       process.exitCode = 2;
       return;
       ```
       Do NOT throw (skill caller should still resolve cleanly).

    2. **Ink component** — function component `<Dashboard cwd={...} />` using React hooks:
       - `useState<ActiveWork>` initialized from the first synchronous read (call `readActiveWork` in a `useEffect` before first render; show "loading" until it resolves)
       - `useEffect` sets up a `setInterval(1000)` polling loop calling `readActiveWork(cwd)`; on successful read, compare `updated_at` with current state, `setState` only if changed
       - `useEffect` cleanup clears the interval
       - `useInput` from Ink to handle `q`, `Q`, `Escape`, and Ctrl+C → call `exit()` from Ink's `useApp`
       - Error state: if `readActiveWork` throws (shouldn't happen per its contract, but defensive), surface a red 1-line error at the bottom but keep polling

    3. **Layout sections** (match CONTEXT `<specifics>` preview):
       - Header: `SUNCO Dashboard · v{version}` (pull version from `packages/cli/package.json` via import or from a const; for this plan, hardcode `v1.3` and leave a TODO for version injection)
       - Active Phase: `▶ Phase {id} ({slug}) · {state} · [{category}]` with `plan {plan_id} · step {current_step}` on the next line. Skip section entirely if `active_phase === null`.
       - Background work: `⚙ Background work ({count})` header. D-14 rules apply (max 3, running + completed within 30 min). Each row: `{kind} ({agent_id_short}) {description} · {state} · {relative_time}`. Skip section if filtered list is empty.
       - Blocked: `⚠ Blocked` header. Show `reason` and `since {relative}` if `blocked_on !== null`, else `(none)`.
       - Next: `→ Next` header. Show `{command} · {reason}` from `next_recommended_action` if non-null, else `(no recommendation)`.
       - Recent skill calls: `⏱ Recent skill calls` header. Show last 3 entries from `recent_skill_calls`: `{skill} · {duration} · {relative}`. Skip if empty.
       - Footer (`StatusBar`): `updated {relative} · polling 1s · q to quit`. If a transient read error is active, append ` · ⚠ read error`.

    4. **Terminal width fallback** — use `process.stdout.columns`. If `< 80`, render a compact 4-line variant (see CONTEXT `<specifics>`). Otherwise, full layout.

    5. **Exit cleanup** — clearInterval + Ink `unmount()` + resolve the outer promise. The function MUST not leave a hanging setInterval behind after promise resolution.

    6. **Imports** — use Ink primitives from `@sunco/core/ui` (if exported via barrel) or direct relative paths. ESM `.js` suffix in all local imports.

    7. **Forbidden:**
       - No writes to `.sun/active-work.json` or any other state
       - No calls into `writeActiveWork` or `appendBackgroundWork`
       - No `ctx.run()` or any skill invocation (TUI is read-only)
       - No chokidar / file watcher
       - No HTTP server / network calls
       - No OMO agent-zoo names (Sisyphus, Hephaestus, Prometheus, Oracle, Librarian)
       - No new `/sunco:*` commands
  </action>
