<objective>
Make the `--max-turns` CLI argument configurable per AgentRequest instead of hardcoded to 1. Layer 6 (cross-model verification) and Layer 1 experts can then use higher values for investigative verification while simple queries remain at 1 turn.

Requirements fulfilled: VCLEAN-04
Phase goal contribution: Enables verification agents to use tools (read files, run tests) instead of being limited to single-turn opinion responses.
Complexity: S
</objective>

  <name>Add maxTurns to AgentRequest and wire in ClaudeCliProvider</name>
  <files>
    - packages/core/src/agent/types.ts

  <read_first>
    - packages/core/src/agent/types.ts
    - packages/core/src/agent/providers/claude-cli.ts
  </read_first>
  <action>
    **types.ts changes:**
    Add to `AgentRequest` interface after the `signal?: AbortSignal` field:
    ```ts
    /** Maximum agent turns (tool-use cycles). Default: 1 for simple queries. */
    maxTurns?: number;
    ```

    **claude-cli.ts changes:**
    In `buildArgs` method, change:
    ```ts
    const args: string[] = ['-p', '--output-format', 'json', '--max-turns', '1'];
    ```
    to:
    ```ts
    const turns = String(request.maxTurns ?? 1);
    const args: string[] = ['-p', '--output-format', 'json', '--max-turns', turns];
    ```

    **claude-cli.test.ts changes:**
    Add or update tests:
    1. Test that `buildArgs` with no `maxTurns` in request produces `--max-turns 1`
    2. Test that `buildArgs` with `maxTurns: 5` produces `--max-turns 5`
    3. Test that `buildArgs` with `maxTurns: 0` produces `--max-turns 0` (edge case)

    Note: `buildArgs` is private. Test via the public `execute` method by mocking execa and inspecting the args passed, OR change `buildArgs` visibility to `protected`/package-private for testability. Preferred: use `as any` cast or extract to a module-level function.

    Alternative approach if buildArgs stays private: test indirectly by verifying the execa call args in the execute mock.
  </action>
