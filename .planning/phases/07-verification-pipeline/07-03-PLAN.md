- `id: 'workflow.validate'`
- `kind: 'deterministic'` (no agent dispatch -- pure coverage parsing)
- `description: 'Run test coverage audit and produce structured report'`
- Options: `--threshold <number>` (minimum line coverage %, default 80), `--phase <number>` (optional, for phase-scoped validation)

Execute flow:
1. Determine coverage output directory: `.sun/coverage` (keep coverage output in .sun/ to avoid polluting project)
2. Spawn vitest as child process via `execFile` from `node:child_process` (promisified). Command: `npx vitest run --coverage.enabled --coverage.provider=v8 --coverage.reporter=json-summary --coverage.reportsDirectory=<coverageDir>`. Timeout: 120_000ms. CWD: ctx.cwd. (Pitfall 6: must spawn as child process, not programmatic)
3. Read `<coverageDir>/coverage-summary.json`. If file missing (Pitfall 3), return failure with clear error: "Coverage output not found. Ensure @vitest/coverage-v8 is installed."
4. Load previous snapshot from state: `ctx.state.get('validate.lastSnapshot')`
5. Parse with `parseCoverageSummary(content, previousSnapshot)`
6. Save current overall as snapshot: `ctx.state.set('validate.lastSnapshot', report.overall)`
7. Format report as markdown:
```
# Coverage Report
Overall: XX.X% lines | XX.X% branches | XX.X% statements | XX.X% functions
Delta: +X.X% lines | ... (if previous snapshot exists)
Threshold: XX% (--threshold)

## Uncovered Files
- path/to/file.ts (0% lines)

## Per-File Coverage
| File | Lines | Branches | Statements | Functions |
|------|-------|----------|------------|-----------|
| ... | XX% | XX% | XX% | XX% |
```
8. Write report to phase directory or stdout
9. Determine pass/fail: overall.lines.pct >= threshold
10. Return SkillResult with success based on threshold, data = CoverageReport
